# 茅台批价追踪 — 项目经验

> 最后更新：2026-06-12

## 一、数据源实测结论

| 来源 | 状态 | 能拿到什么 |
|------|:--:|------|
| **今日酒价公众号 (mp.weixin.qq.com)** | ⭐⭐⭐⭐⭐ | 最权威，baoyu-fetch 可用，每日散瓶+原箱配对 |
| **财联社(cls.cn)** 引用今日酒价 | ⭐⭐⭐ | 散瓶+原箱配对，行业最可靠 |
| **搜狐酒价周报** | ⭐⭐⭐ | 每周逐日 breakdown |
| **云酒网** | ⭐⭐ | 单日实时报价 |
| **金价查询网** huangjinjiage.cn | ⭐⭐ | 多年份对比表 |
| **winesinfo.net** | ⭐⭐ | 单页逐日，URL 含日期但 ID 不可预测 |
| **界面新闻/易茅时价** | ⭐ | 转载，需交叉验证 |
| **jujindata.com** | ⚠️ | 有逐日表但数据是复合指数(1660/1670)，**不是批价** |
| **meijiu.com、maotai.wgnds.com** | ❌ | 不可达 |

## 二、数据结构

```json
{
  "date": "2026-01-01",
  "sanping": 1490,
  "yuanxiang": 1660,
  "guide_price": 1499,
  "signal": "🔴",
  "source": "财联社",
  "note": "备注"
}
```

## 三、信号规则

| 信号 | 条件 | 含义 |
|:--:|------|------|
| 🔴 | 散瓶 < **当期指导价** | 渠道亏损 |
| 🟡 | 指导价 ~ 1800 | 正常区间 |
| 🟢 | > 1800 | 溢价区间 |

> 指导价：¥1,499 (1/1~3/30) → ¥1,539 (3/31起)。`regenerate.py` 自动判断。

## 四、更新流程

**日常更新**（自动化，每天 14:00）：
1. 优先源：如果用户提供今日酒价公众号链接 → baoyu-fetch 直抓
2. 兜底：WebSearch 搜索财联社/云酒网等二手源
3. 去重 → 追加 data.json → `python regenerate.py` → `git push`

**批量回填**（一次性，用 light 模式）：
```bash
cd xhs-bulk-download-project && node batch_extract_all.mjs <links.json> <data.json>
```
从今日酒价专辑页提取全部历史价格数据，同时输出 data.json（茅台散瓶/原箱）+ all_prices.jsonl（全品类）。

## 五、输出文件结构

| 文件 | 用途 |
|------|------|
| `data.json` | 茅台散瓶/原箱数据源（131点，覆盖1月~6月） |
| `all_prices.jsonl` | 全品类酒价（~2.4万行，251款酒×110天） |
| `regenerate.py` | 改 data.json 后必跑 |
| `batch_extract_all.mjs` | light模式批量提取脚本 |
| `2026/2026-{MM}.md` | 各月明细 |
| `2026总览.md` | 年度全表 |

## 六、覆盖率现实

- **当前**：131 个数据点，覆盖 2026-01-01 ~ 2026-06-12（不含春节/周末缺口）
- **之前**：42 点/181天 ≈ 23%
- **现在**：131 点，散瓶+原箱配对率大幅提升
- **all_prices.jsonl**：109 天完整全品类数据（1月~4月）
- 结论：今日酒价公众号是行业最权威源，baoyu-fetch 直抓可行

## 七、GitHub 推送踩坑

| 问题 | 解决 |
|------|------|
| SOCKS/HTTP 代理 → git TLS 握手失败 | 切 TUN/虚拟网卡模式 |
| gh api 也需不走代理 | `http_proxy="" https_proxy=""` |
| git push 不通时 | 用 `gh api` 四步推送：blob → tree → commit → ref update |
| remote URL 缺 token | `https://ailubo:TOKEN@github.com/ailubo/maotai-prices.git` |
| 本地 master ≠ 远程 main | `git push origin master:main` |

## 八、飞书文档经验

- **不要用 convert API**：块顺序乱、表格不兼容
- **手工构建 block 对象**可靠：`{block_type, heading2/heading3/text/bullet, elements, style}`
- **不兼容的块**：divider(9)报错、list_item(13)需父容器、table(31/32)不可用
- **清空重写**：`batch_delete start_index=0 end_index=n-1`
- **逐个插入**比批插入稳：`index: -1` 追加到末尾
- **权限**：`POST /permissions/{docId}/members?type=docx` with `perm: full_access`
